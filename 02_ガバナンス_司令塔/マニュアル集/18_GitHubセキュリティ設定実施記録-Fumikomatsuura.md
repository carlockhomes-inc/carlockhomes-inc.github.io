# GitHubセキュリティ設定 実施記録

## 1. 実施概要
- **実施日**: 2026年4月25日
- **実施者**: カーロックホームズ社 管理者
- **目的**: CarlockHomesプロジェクトのGitHubリポジトリにおける安全性の強化、および情報漏洩リスクの低減。

## 2. 実施項目と判断根拠

### ① リポジトリ公開設定の確認
- **現状**: 公開（Public）
- **判断**: **「公開」を維持**
- **理由**: 
    - 現在のGitHubプランが「Free（無料版）」であるため。
    - 無料版でリポジトリを「非公開（Private）」に設定すると、GitHub PagesによるHP公開が停止（404エラー）するため。
    - ソースコード内に機密情報（APIキー、個人情報等）が含まれていないことを確認済みであり、公開状態でも実質的なセキュリティリスクは極めて低いと判断。

### ② プランの確認
- **組織アカウント**: `carlockhomes-inc` -> **GitHub Free**
- **個人アカウント**: `fmfmfm0207-max` -> **GitHub Pro (Payment info configured)**
- **備考**: 今後、ソースコードを完全に隠蔽しつつHPを公開し続ける必要がある場合は、組織アカウント（carlockhomes-inc）の有料プラン（GitHub Team）へのアップグレードを検討する。

### ③ 2段階認証（2FA）の導入
- **設定対象**: 個人アカウント（fmfmfm0207-max）
- **設定手法**: 認証アプリ（Authenticator app）によるワンタイムパスワード
- **実施結果**: **設定完了**
- **効果**: アカウントの乗っ取りリスクを最小化。組織（carlockhomes-inc）に対するすべての操作もこの2段階認証によって保護される。

## 3. 保守・管理上の注意
- **リカバリーコードの管理**: 2段階認証設定時に発行された「Recovery codes」は、デバイス紛失時の唯一の復旧手段として、安全な場所にオフライン保存・バックアップ済み。
- **機種変更時の注意**: スマホの機種変更時には、認証アプリの設定移行を忘れないこと。
- **2FAの継続**: GitHubの全ユーザーに対する2FA義務化（2026年6月8日期限）に先んじて対応を完了。

## 4. 承認・確認
本記録は、2026年4月25日時点の最適解として実施されたものである。
