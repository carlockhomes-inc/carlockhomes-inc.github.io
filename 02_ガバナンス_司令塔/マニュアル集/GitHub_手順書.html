目的

GitHub リポジトリに重大な障害やセキュリティインシデントが発生した際、速やかにサービスを止め、影響範囲を最小化する手順を示す。

前提条件

  • 管理者アカウントでログインできること
  • リポジトリのバックアップが取得できる権限があること
  • 2段階認証（2FA）が有効化されていること（※手順 2 参照）

手順

  1. バックアップ取得
    対象リポジトリをローカルにクローンし、全ブランチ・タグを保存する。
  2. 2FA（二要素認証）の設定
    個人アカウントおよび組織アカウントの 2FA を有効化。設定が未完了の場合は Settings → Security → Two-factor authentication で有効化。
  3. リポジトリの公開設定確認
    「Settings → General → Repository visibility」で Public か Private を確認し、緊急時は Private に変更（※GitHub Pages が必要な場合は一時的に Public に戻す）。
  4. 緊急用静的ページの配置
    緊急対応用の static HTML（例: emergency.html）をリポジトリの gh-pages ブランチにコミットし、GitHub Pages の Source を gh-pages に設定。
  5. アクセス遮断
    必要に応じて Settings → Branch protection rules で全ブランチへのプッシュを禁止し、CI/CD パイプラインを一時停止する。
  6. 社内外への連絡
    障害発生時は速やかに担当者へメールまたは電話で報告し、復旧予定時間を共有する。
  7. 復旧手順
    障害が解消したら、バックアップから復元または正常ブランチへマージし、再度公開設定と CI/CD を有効化する。

注意点

  • 2FA が未設定の場合、アカウント乗っ取りリスクが高まります。
  • リポジトリを Private に変更すると、GitHub Pages が停止するため、緊急用ページは別ブランチで管理してください。
  • バックアップは定期的に外部ストレージへ保存し、復元手順を事前に検証しておくこと。
