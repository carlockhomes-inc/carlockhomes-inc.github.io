# 📝 統合ドキュメント自動生成システム Blueprint

## 1. 概要 (Overview)
「案件データ」を1箇所に入力するだけで、同じデータ群を利用して「見積書」「報告書（モリモト形式等）」「請求書」の3つのドキュメントを、Googleスプレッドシート上で自動的に生成・出力するシステム。

Excel運用で発生していた「数式が壊れる」「手入力の手間とミス」といった課題を完全に解決し、会社のPCスペックに一切依存せずクラウド上でサクサク動く業務基盤を構築する。

## 2. 既存資産の活用 (Eco-system)
- **MQ業務管理システム（先日開発）のデータ同期ロジック**を流用。
- 「マスターシート」の行データを元に、「指定したレイアウトの別シート（テンプレート）」へ値をマッピングするGASの仕組みを再利用することで、短期・低コスト・安全なスピード開発を実現。

## 3. MVP : Day 1の最小必須機能 (Minimum Viable Product)
以下の機能のみを「Day 1」の完了条件とする。多機能化（Webアプリ化やメール自動送信など）はPhase 2以降とする。

1. **データマスター（DB）の統合**: 全ての案件情報や作業費用の明細を1つの「案件一覧（マスター）シート」に入力する。
2. **絶対に壊れない出力層**: スプシ内に「見積書」「報告書」「請求書」の独立したフォーマット用シートを用意し、ここは**「編集不可（保護状態）」**とする。
3. **ワンクリック生成処理（GAS）**: マスターシートで「出力したい案件」を選び（チェックボックス等）、システムメニューから「ドキュメント生成」を押すと、3つのテンプレートに値が流し込まれる。
4. **PDF自動保存（ドライブ連携）**: 生成された3点セットは、自動的にGoogleドライブ内の指定フォルダに「PDF」として保存されるか、または印刷用に見やすく表示される。

## 4. アーキテクチャ構成
本システムは会社のPCのスペック制約を受けないため、完全なGoogle Workspaceクラウドネイティブ構成とする。

- **Database（データ蓄積）**: Googleスプレッドシート（案件マスターシート）
- **Frontend（操作画面）**: Googleスプレッドシート（案件マスターの行入力部分のみをユーザーに開放）
- **Backend（処理ロジック）**: Google Apps Script (GAS)
- **Storage（ファイル保管）**: Google Drive（必要に応じて生成されたPDFを案件ごとに自動保存）

## 5. データフローと構造 (Data Structure)

### シート構成
1. **案件マスター（DB）シート**
   - 受付No, 顧客名（管理組合名等）, 住所, 連絡先, 依頼者名, 作業担当者, 依頼内容サマリー
   - 明細1（品名/数量/単価）, 明細2, 明細3...
   - 備考
2. **⚙️設定シート（マスタ）**
   - 消費税率（10%）、自社情報（カーロックホームズの住所・電話番号など固定値）
3. **📄【出力用】見積書テンプレート（編集ロック）**
4. **📄【出力用】報告書テンプレート（編集ロック）** 
   - ※今回はご提示いただいたモリモト書式を完全再現
5. **📄【出力用】請求書テンプレート（編集ロック）**

### アクションの流れ（ユーザー体験）
1. スタッフが「案件マスターシート」の新規行に、今回の受付内容や明細データを入力する。
2. その行の右端にある「出力チェック」ボックスに✅を入れる。
3. スプシ上部のメニューバーから「システム処理 ＞ ドキュメント一括PDF化」を実行。
4. GASが動き、指定された行のデータを読み取る。
5. 3つの「出力用テンプレートシート」の所定のセルにデータを自動入力する。
6. 完成した3シートを印刷、もしくはPDFとして書き出し。
7. 「出力完了」のメッセージが出る。（スタッフはそれを送信・提出するだけ）

## 6. プロからの引き算・制約の提案（悪魔の代弁者）
- スプレッドシート内で複数の「画像（現場のBefore/After写真等）」を各セルに動的に貼り付ける処理は、運用を続けるとシート全体が非常に重くなり破損の原因になります。
- そのため、**Day 1では「報告書は今回ご提示いただいた文字・数字情報のみのフォーマット」とし、「画像処理」はシステムに含めない**という安全な制約を設けます。画像が必要な場合は、別途フォルダにまとめる等の運用回避を推奨します。
