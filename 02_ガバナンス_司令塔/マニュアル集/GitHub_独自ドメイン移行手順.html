🌐 GitHub Pages 独自ドメイン移行手順
（法人アカウント・完全版）

現在個人のGitHubで作成しているホームページを、会社用のGitHubアカウントへ移行し、さらに旧サーバーで使っている carlockhomes.co.jp を繋ぎ直すための完全版手順です。

フェーズ1：会社用GitHubアカウントの開設とデータ移行

まずは、個人のGitHubにあるHPデータを、会社用の公式アカウントに移します。

  1. 会社用アカウントの作成
    GitHubのトップページから、会社用のメールアドレスを使用して新しいGitHubアカウントを開設します。
  2. リポジトリ（保管庫）の移行
    現在の個人アカウントにあるHPの保管庫を、新しく作った会社用アカウントへ移譲するか、コードをコピーして新しい保管庫としてアップロードします。

フェーズ2：GitHub側の設定（カスタムドメインの登録）

会社用アカウントにHPデータが無事に移ったら、ドメインを受け入れる準備をします。

  1. 会社用アカウントにログインし、移したリポジトリを開きます。
  2. 右上の歯車マーク [Settings] をクリックします。
  3. 左側のメニューから [Pages] をクリックします。
  4. 画面を下へスクロールし、[Custom domain] の入力欄を見つけます。
  5. 枠の中に carlockhomes.co.jp と入力し、[Save] をクリックします。

※「DNS check in progress」といった表示が出ますが、この時点ではエラーになっても気にしないでください。

フェーズ3：ドメイン側の設定（DNSレコードの変更）

ここが一番重要な、ドメインの「案内先」を旧サーバーからGitHubへ切り替える作業です。
ドメインを管理している会社の管理画面にログインし、「DNSレコード設定」にて以下の4つのAレコードを登録します。

  • ホスト名(サブドメイン): 空白（または @ ）
  • タイプ: A
  • 値（IPアドレス）:
    1. 185.199.108.153
    2. 185.199.109.153
    3. 185.199.110.153
    4. 185.199.111.153
⚠️ ご注意
もし、すでに旧サーバーに向けられた別のAレコードが存在している場合は、それを削除してから上記の4つを追加するか、上記のIPアドレスに上書きしてください。

フェーズ4：SSL（HTTPS）の有効化

DNSレコードの設定を保存したら、インターネット全体に新しい住所が反映されるまで数時間〜最大24時間程度かかります。

  1. しばらく待ってから、再度 会社用GitHub の [Settings] > [Pages] 画面を開きます。
  2. 「DNS check successful」と緑色のチェックマークがついていることを確認します。
  3. 同じ画面にある [Enforce HTTPS] というチェックボックスにチェックを入れます。
🎉 完了
これで、安全な https://carlockhomes.co.jp での表示が完了します！

付録2：よくあるご質問（Q&A）と用語解説

Q. ドメインを移動させるためには何が必要ですか？

A. 必要なものは大きく2つだけです。「ドメインを管理しているサービス（お名前.comなど）のログイン情報」と、「その管理画面でのDNSレコードの書き換え作業」です。

Q. 個人アカウントから会社用へHPをお引っ越しさせると、IPアドレスも新しいものに変わりますか？

A. いいえ、IPアドレスは変わりません！マニュアルに記載されている4つのIPアドレスは、GitHub全体の「総合受付」のようなものです。受付から先の「どの部屋（アカウント）のデータを表示するか」は、GitHubの管理画面でドメインを登録しておくことで自動的に行われます。そのため、どのGitHubアカウントを使ってもIPアドレスは共通です。

Q. ドメインが変わってもちゃんとHPデータにたどり着けるのが不思議です。

A. インターネットの「電話帳」の仕組み（DNS）のおかげです！ドメイン管理会社が電話帳の役割をしており、「このドメインの住所はGitHub（185.199...）だよ」と案内します。そして到着したGitHub側でも「このドメインの表札が掛かっているのはここだね」と判断してくれるため、裏側でどんな引っ越しをしても迷子にならずにHPが表示されます。

Q. 依頼メールにある「wwwありの設定（CNAME等）」についての一文は、どういう意味ですか？

A. インターネットの世界では、carlockhomes.co.jp（wwwなし）と www.carlockhomes.co.jp（wwwあり）は「別の住所」として扱われます。もし「wwwなし」の引っ越しだけをして終わってしまうと、「wwwあり」でアクセスしたお客さんがエラーになってしまいます。あの一文は、管理会社のプロに対して「お客さんがどちらを入力してもちゃんと表示されるように、プロの判断でよしなに調整しておいてくださいね」と丸投げするための魔法の言葉です。

📚 専門用語の解説

ドメイン
インターネット上の「住所」や「表札」のこと。（例：carlockhomes.co.jp）
IPアドレス
インターネット上の「実際の住所（緯度経度のような数字の羅列）」。（例：185.199...）
DNS（ディーエヌエス）
ドメイン名とIPアドレスを結びつける「インターネットの巨大な電話帳」の仕組み。
Aレコード（エーレコード）
電話帳（DNS）の中で、「このドメインの実際のIPアドレスはこれですよ」と直接指定するための設定項目。
リポジトリ
GitHub上にある「データの保管庫（フォルダ）」のこと。ここにHPのデータが入っています。
GitHub Pages（ギットハブ・ページ）
GitHubの保管庫に置いたデータを、そのまま世界中に「ホームページ」として公開できる無料のサーバー機能のこと。
