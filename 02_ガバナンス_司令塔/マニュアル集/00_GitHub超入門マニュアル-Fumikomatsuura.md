## 📖 1. GitHub（ギットハブ）とは？
ひとことで言うと、**「プログラミング専用の、超優秀なセーブデータ保管庫（ネット上の金庫）」** です。
ゲームで「ボス戦の前」にセーブするように、プログラムも好きなタイミングで名前を付けてセーブできます。そして、そのセーブデータをインターネット上に預かってくれるのが「GitHub」です。

---

## 🚀 2. GitHubの登録方法（無料）
最初の登録はたったの3ステップで終わります！（※すでに登録済みの場合はスキップしてOKです）

1. **公式サイトを開く**
   ブラウザで [GitHub公式サイト（https://github.com/）](https://github.com/) にアクセスします。
2. **「Sign up（サインアップ）」を選ぶ**
   画面の右上にある「Sign up」ボタンを押します。
3. **順番に入力する**
   画面の案内に従って、以下の3つを入力すれば完了です！
   - メールアドレス （普段使っているものでOK）
   - パスワード （破られにくいものを設定）
   - ユーザー名 （ネット上で表示されるニックネーム、英数字）

---

## ⚙️ 3. GitHubでできること（3つの最強機能）

### ① 🛡️ ネット上の金庫（バックアップ機能）
パソコンが突然壊れたり、最新のパソコンに買い替えたりしても大丈夫。セーブデータは「ネット上の金庫」にあるので、ログインしてダウンロードすれば、いつでも続きから作業を再開できます。

### ② ⏱️ タイムマシン（履歴管理機能）
「あれ？変な画面になっちゃった！さっきのデータを消さなきゃよかった…」となっても安心です。
GitHubは過去すべてのセーブを残しているので、「昨日の夜の状態」や「1週間前の状態」に一瞬で巻き戻す（タイムマシン）ことができます。

### ③ 🤝 協力プレイ（チーム開発・共有機能）
友達や仕事のメンバーと一緒に同じアプリを作れます。
同じプログラミングファイルを同時に編集しても、お互いのデータが消えないように「自動でガッチャンコ（合体）」してくれるすごい機能を持っています。

## 💾 4. データの呼び出し方（保存と復元）

基本的には**「私（AI）にチャットでお願いする」**のが一番カンタンで確実ですが、ここでは代表的な「魔法の呪文（コマンド）」を紹介します。

### ■ パターンA：「よし、今日の進捗をセーブして金庫に預けよう！」
作成したファイルや変更をGitHubに預ける（Push：プッシュ）ときの呪文です。
```bash
# 1. 変更したものを全部集める
git add .

# 2. セーブデータに名前をつける（例：新しい機能を追加したよ）
git commit -m "新しい機能を追加したよ"

# 3. ネット上の金庫（GitHub）に送信する
git push
```
💡 **AIへの頼み方：「今日の作業分をGitHubに保存して！」**

### ■ パターンB：「新しいパソコンを買った！データを丸ごと持ってこよう！」
まったく新しいパソコンで、最初からすべてのファイルと歴史（セーブデータ）をダウンロード（Clone：クローン）するときの呪文です。
```bash
# ※あなたの実際のURLを入れてあります！
git clone https://github.com/fmfmfm0207-max/m-lab.git
```
💡 **自分のURLの確認方法：**
ブラウザでご自身の金庫のページ（[https://github.com/fmfmfm0207-max/m-lab](https://github.com/fmfmfm0207-max/m-lab)）を見ると確認できます。もちろん私（AI）に「私のURLって何だっけ？」と聞いていただいてもOKです！
💡 **AIへの頼み方：「新PCにデータを丸ごとダウンロードする呪文を教えて！」**

### ■ パターンC：「家のパソコンで進めた分を、このパソコンに引き継ごう！」
すでにデータはあるけれど、他のパソコンで進めた「最新の差分」だけを引っ張ってきたい（Pull：プル）ときの呪文です。
```bash
git pull
```
💡 **AIへの頼み方：「最新のデータをGitHubから引っ張ってきて！」**

### ■ パターンD：「やばい！昨日動いていた時点のセーブデータに戻したい！」
過去のセーブを呼び出す呪文（少し複雑です）。
💡 **AIへの頼み方：「〇〇がおかしくなったから、昨日の状態に戻して（復元して）！」** ← これが一番オススメの呼び出し方です！

## 💰 5. よくある「お金のギモン」

**Q.「他のエンジニアに手伝ってもらうと、追加でお金がかかるの？」**

結論から言うと、**システム利用料は一切かからず、完全無料（0円）で使えます！**

- **何人招待しても無料！**
  「私の金庫（データ）にAくん達も入っていいよ！」と招待して一緒に作業しても、GitHubから追加料金を請求されることは絶対にありません。
- **知らない人が勝手に入ってくることはない！**
  金庫には「カギ（アクセス権）」がしっかりかかっています。知らないエンジニアが勝手に入ってきて、あとでお金を請求される…なんていう詐欺のようなことも100%ないので安心してください。
- **もしお金がかかるとしたら？**
  - GitHubに払うお金ではなく、「手伝ってくれたエンジニア本人にお給料（お礼）を払う」時
  - 何百人もいるような超大企業向けの「特別な機能（有料プラン）」を使いたい時（個人のビジネスや一般的なチーム開発なら一生無料プランのままで全く問題ありません！）

## 🆘 6. トラブルシューティング（パスワードを忘れたら？）

GitHubのパスワードを忘れてしまった場合は、以下の2つの方法で確認・再設定ができます。

### ① ブラウザに保存されているか確認する（一番カンタン）
一度でもログインしたことがあれば、普段お使いのWebブラウザにパスワードが記憶されている可能性が高いです。

- **🌐 Google Chrome の場合:**
  1. 右上の「**︙**」をクリック ＞「**Googleパスワードマネージャー**」を開く。
  2. 検索窓で「GitHub」と検索し、目のマーク（👁️）を押すとパスワードが見られます。
- **🌐 Microsoft Edge の場合:**
  1. 右上の「**…**」をクリック ＞「**設定**」を開く。
  2. 左メニュー「**プロファイル**」＞「**パスワード**」から「GitHub」を探し、目のマーク（👁️）を押します。

### ② パスワードを再設定する（どこにもない場合）
どうしても見つからない場合は、新しく設定し直す（リセットする）のが一番早いです。

1. GitHubのログイン画面（Sign in画面）を開く。
2. パスワード入力欄の下にある **「Forgot password?」**（パスワードをお忘れですか？）をクリック。
3. 登録しているメールアドレスを入力して送信すると、新しいパスワードを決めるための案内メールが届きます。

💡 **AIへの頼み方：「作業中に急にパスワードを求められた！」**
VS Codeなどの作業中に急に認証エラーが出たときは、パソコン内の設定（資格情報マネージャーなど）が原因のことが多いです。その時は私（AI）にそのままお声がけください！一緒に解決します！

## 🖥️ 7. GitHubのWeb画面（サイト）の見かた

お見せいただいた画像のように、ブラウザで自分の金庫（リポジトリ）を開いたときによく使う、**「これだけ知っていればOK」な3つのポイント**をご紹介します。

### ① 真ん中のリスト（ファイル一覧）
現在パソコンの中に保存されているファイルやフォルダが、そのままネット上にも保管されているのが見えます。
スマホや他のPCからでも、ここをクリックすればファイルの中身（コードや文章）をサクッと確認できます。

### ② 履歴ボタン（🕒 〇〇回のコミット）
ファイル一覧の右上に小さく出ている時計のマークです。（画像だと「🕒 50回のコミット」と表示されています）
ここを押すと**「タイムマシン（過去のセーブ履歴一覧）」**の画面になり、「いつ、どんなセーブ名で保存したか」が上から新しい順にずらーっと並びます。過去に何をいじったか確認したい時に便利です。

### ③ 緑色の「<> コード」ボタン
ファイル一覧の右上にある目立つ緑色のボタンです。主に2つの使い方をします。
- 新しいパソコンへ**データを丸ごとダウンロード（Clone）するためのURL**をコピーするとき。
- 魔法の呪文（コマンド）を使わずに、そのまま「ZIPファイル形式（ZIPをダウンロード）」でパソコンにまるごと保存・解凍したいとき。

> 💡 **豆知識：日本語表示について**
> 画面に「メイン」や「支部（branchの直訳）」といった言葉が表示されている場合、ブラウザ（ChromeやEdge）の**自動翻訳機能**がオンになっています。IT用語が無理やり日本語に翻訳されると少しややこしいため、余裕が出てきたらブラウザのURLバー右側にある翻訳ボタンから「英語のまま表示（翻訳しない）」設定で使うのが、エンジニア界隈ではオススメです！

## 🌐 8. おまけ：無料で自分のWebサイト（お店）を公開しよう（GitHub Pages）

GitHubには、自分が作ったプログラム（HTMLなど）を、そのまま**「本物のWebサイト」として世界中に無料で公開できる「GitHub Pages（〇〇.github.io）」**という神機能があります。

### GitHub Pagesのイメージ
- **通常のGitHub：** 作業途中の絵や設計図をしまっておく「ネット上の金庫（倉庫）」
- **GitHub Pages：** 倉庫の中にある「完成した絵（ホームページ）」を、表の「**ショーウィンドウ（展示室）**」に飾って、世界中の人にいつでも見てもらう機能。この仕組みも**ずっと無料（サーバー代0円）**です！

### 「お店（ネットショップ）」は作れるの？
**「商品のカタログ（ショーウィンドウ）」なら作れますが、「レジ（決済システム）」は別のサービスを組み合わせる必要があります。**

GitHub Pagesはあくまで「展示（文字や画像を見せるだけ・安全性を重視）」に特化しているため、これ単体でショッピングカートやクレジットカード決済のような「裏側での複雑な計算処理（データベース）」を持たせることはできません。

**💡 具体的にどうやってお店を作るのか？（よくある手法）**
1. GitHub Pagesで、商品写真や説明が載った「**めちゃくちゃオシャレなホームページやLP（カタログ）**」を作る。
2. そのページの「購入する」ボタンを押すと、**「Stripe（決済専用サービス）」**や**「BASEなどの既存ショップシステム」**の支払い画面に飛ぶようにリンクを貼る。
3. これだけで、**「維持費0円で運営できる自分だけのお店」**の完成です！小規模な販売やポートフォリオサイトとしては最強の組み合わせです。
