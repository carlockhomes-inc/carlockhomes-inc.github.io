# 00_MLab_業務管理システム設計図 (Business Management System Blueprint)

## 1. プロジェクト概要
受注（メール/電話/FAX）から、現場での在庫消費、完了報告、そして最終的なPDF請求書発行までをPC中心に一括管理する「M-Lab 業務OS」の設計図。

- **コンセプト**: 「事務はPCで効率的に。現場はスマホで直感的に。」
- **ターゲット**: 事務担当者（PC管理）、現場担当（スマホ入力）
- **主要な課題**: 受注データの多重入力の廃止、請求書作成の自動化、リアルタイムな在庫把握。

## 2. システム構成（アーキテクチャ）
Googleエコシステムを最大限に活用し、低コストかつ高拡張な構成とする。

| コンポーネント        | 役割                                      | デバイス   |
| :-------------------- | :---------------------------------------- | :--------- |
| **Google Sheets**     | データベース・管理台帳・ダッシュボード    | **PC**     |
| **AppSheet**          | 現場での在庫スキャン・報告入力            | **スマホ** |
| **Apps Script (GAS)** | PDF請求書自動生成・通知・在庫計算ロジック | -          |
| **Google Drive**      | 顧客マスタ・成果物PDFの保存               | -          |

## 4. 業務フロー

### Step 1: 受注 (PC)
1. 事務所に電話/FAX/メールが届く。
2. 管理者が **PC上の受注台帳** に内容を入力し、ステータスを「受注済」にする。
   - この時点で現場で確認できる状態になる。

### Step 2: 現場対応 & 在庫消費 (スマホ)
1. 現場スタッフが **AppSheet** で担当案件を開く。
2. 使用する部材を **スマホカメラでスキャン** し、数量を入れる。
   - 在庫が減り、案件の見積金額に即座に反映される。
3. 現場写真を数枚撮って保存。

### Step 3: 完了確認 & 請求書発行 (PC)
1. 事務所で状態が「完了」になったのを確認。
2. **PCのスプレッドシート** 上の「請求書発行」ボタンをクリック。
3. **Google Apps Script** が起動し、以下の処理を実行：
   - テンプレートにデータを差し込み。
   - **PDF生成**。
   - Googleドライブの顧客別フォルダに保存。
   - (オプション) クライアントにメール送信。

## 5. MVP（最初の一歩）の受け入れ条件
- [ ] PCのスプレッドシートに案件を入れたら、スマホのAppSheetで表示されること。
- [ ] スマホで部材をスキャンしたら、スプレッドシートの在庫数が減ること。
- [ ] ボタン一発で、サンプルの請求書PDFが生成されること。

## 6. プロの提案：Day 2 以降の拡張
- **OCR導入**: FAXの画像をAIに読み込ませ、自動で「受注台帳」の下書きを作る。
- **カレンダー連携**: 受注日時に合わせて、Googleカレンダーに自動で予定を確保する。
- **支払い管理**: 入金確認チェックボックスを設け、未回収金の自動通知を行う。
