# 🐙 06. GitHubの使い方 (初級編：個人倉庫を作ろう)

さあ、いよいよ世界中のエンジニアの仲間入りです！
まずは、難しいコマンドは置いておいて、**「GitHubのWebサイト」**で何ができるかを知りましょう。

## 🏗️ Step 1: アカウントを作る（会員登録）
まずは、GitHubという「巨大倉庫」の会員証を作りましょう。

1.  **[GitHub.com](https://github.com/)** にアクセスする。
2.  「Sign up」から、メールアドレスなどを入れて登録する。
    *   **注意**: 英語のサイトですが、怖がらないで！「倉庫の契約書」だと思えばOK。
3.  **ユーザー名 (Username)** を決める。
    *   これが世界に公開されるあなたの名前です。かっこいいのをつけましょう！
    *   （例: `omatsu-san` とか `fumiko-design` とか）

## 📦 Step 2: リポジトリ（Repository）を作る
「リポジトリ」＝ **「プロジェクトごとの専用箱」** です。
例えば、「料理マニュアル」という箱、「家計簿」という箱。

1.  右上の「＋」マークから **「New repository」** を選ぶ。
2.  **Repository name**（箱の名前）を入れる。
    *   例: `my-second-brain-backup`
3.  **Public**（公開）か **Private**（非公開）を選ぶ。
    *   **Private** にしましょう！ おまつさんの大事な脳みそは、まだ秘密にしておきたいですからね。
4.  「Create repository」ボタンをポチッ！
    *   これで、インターネット上に **「おまつ専用の空っぽの箱」** ができました！🎉

## 📤 Step 3: ファイルをアップロードする（手動編）
最初は「コマンド」を使わなくても、Web画面からファイルを置けます。

1.  作った箱の画面にある **「Add file」** → **「Upload files」** を選ぶ。
2.  PCにあるファイルを、ドラッグ＆ドロップで放り込む。
3.  **Commit changes**（変更を記録）のところに、メモを書く。
    *   例: 「初めてのアップロード！」
4.  「Commit changes」ボタンを押す。
    *   これで、「倉庫」におまつさんのファイルが保管されました！

---

## 🦁 なぜこれをやるの？
「Sync（同期）」があるから、持ち運びはできてますよね？（あれで！）
でも、GitHubは **「歴史を残せる」** のが最大の違いです。

*   **Obsidian Sync**: 常に「最新の状態」しか残らない。（上書きされちゃう）
*   **GitHub**: 「昨日のお昼の状態」「1週間前の状態」…全部の歴史が残る。

「あーっ！間違って全部消しちゃった！」
という時でも、GitHubなら**「3日前の世界に戻して！」**ができるんです。
これが**「タイムマシン」**と呼ばれる理由です！🕰️✨

## 🦁 Q&A: よくある質問

### Q. どのPCからでも見れるの？
**A. はい！見れます！** 🌍
GitHubはインターネット上の「巨大倉庫」です。
AmazonやInstagramにログインするのと同じで、
**「IDとパスワード」**さえあれば、**会社のPC**でも、**家のPC**でも、**スマホ**からでも、いつでも自分の箱を開けられます！🔑
